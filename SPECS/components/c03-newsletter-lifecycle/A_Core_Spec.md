# C03 Newsletter Lifecycle вҖ” Core Specification

> Manages the draft вҶ’ published lifecycle for newsletters. Saves generated `.md` drafts to SAP Object Store, converts to `.html` on publish, stores the published file in Object Store, and records all state in HANA Cloud via C05. No AI capabilities.

---

## 1. Purpose

C03 is the custodian of the newsletter lifecycle after generation. It:

- Receives a generated Markdown string from C02 and saves it as a `.md` draft to SAP Object Store; records the newsletter record in HANA Cloud via C05.
- Exposes a list/detail API for the Newsletter tab in the React UI.
- On publish: retrieves the `.md` from Object Store, converts it to HTML via `markdown-it`, writes the `.html` to Object Store, updates the newsletter record in C05 (`status = published`, `published_at`, `object_store_key` updated to HTML key).
- Serves the published HTML directly from Object Store via a signed or proxied URL, or by streaming the Object Store object through Express.
- Supports deletion of a newsletter (removes both Object Store objects and the HANA record).

**Rules:**
- A newsletter starts as `draft` when C02 calls `saveDraft()`.
- `published` is a terminal state вҖ” there is no unpublish action at MVP.
- Deletion is allowed from both `draft` and `published` states.
- A published newsletter's HTML is publicly accessible via `GET /published/:filename.html` вҖ” no API key required.

## 4. Object Store Key Convention

| File type | Object Store key pattern | Notes |
| :-------- | :----------------------- | :---- |
| Draft Markdown | `drafts/<filename>.md` | Written by `saveDraft()`; filename generated by C02 (e.g. `newsletter_2026-05-03`) |
| Published HTML | `published/<filename>.html` | Written by `publish()`; same base filename as the draft |

The `object_store_key` column in the `newsletters` HANA record always reflects the **current primary file** вҖ” `drafts/...` for draft, `published/...` for published.

## 6. Data Flows

### 6.1 Save draft (called by C02 after generation)

```
saveDraft({ filename, topicList, markdownContent })
  вҶ’ S3.putObject({ key: 'drafts/<filename>.md', body: markdownContent })
  вҶ’ C05.insertNewsletter({ filename, topicList, objectStoreKey: 'drafts/<filename>.md' })
  вҶ’ return { id, filename, status: 'draft' }
```

### 6.2 List / detail

```
GET /api/v1/newsletters
  вҶ’ C05.listNewsletters()
  вҶ’ return array of newsletter metadata

GET /api/v1/newsletters/:id
  вҶ’ C05.getNewsletter(id) вҶ’ 404 if not found
  вҶ’ return newsletter record
```

### 6.3 Edit draft content

```
PUT /api/v1/newsletters/:id/content
  вҶ’ C05.getNewsletter(id) вҶ’ 404 if not found; 409 if status = published
  вҶ’ S3.putObject({ key: 'drafts/<filename>.md', body: markdownContent })
  вҶ’ return 200 { id, filename }
```

### 6.4 Preview draft

```
GET /api/v1/newsletters/:id/preview
  вҶ’ C05.getNewsletter(id) вҶ’ 404 if not found; 409 if status = published
  вҶ’ S3.getObject({ key: record.objectStoreKey })
  вҶ’ return { markdownContent: '<string>' }
```

### 6.4 Publish

```
POST /api/v1/newsletters/:id/publish
  вҶ’ C05.getNewsletter(id) вҶ’ 404 if not found; 409 if already published
  вҶ’ S3.getObject({ key: 'drafts/<filename>.md' }) вҶ’ retrieve Markdown
  вҶ’ markdown-it.render(markdownContent) вҶ’ htmlContent
  вҶ’ S3.putObject({ key: 'published/<filename>.html', body: htmlContent, contentType: 'text/html' })
  вҶ’ C05.updateNewsletter(id, { status: 'published', publishedAt: now, objectStoreKey: 'published/<filename>.html' })
  вҶ’ return { id, filename, status: 'published', publishedAt, htmlPath: '/published/<filename>.html' }
```

### 6.5 Serve published HTML

```
GET /published/:filename.html  (unauthenticated)
  вҶ’ C05.listNewsletters() filtered by filename вҶ’ 404 if not found or not published
  вҶ’ S3.getObject({ key: 'published/<filename>.html' })
  вҶ’ pipe S3 object stream to Express response (Content-Type: text/html)
```

### 6.6 Delete

```
DELETE /api/v1/newsletters/:id
  вҶ’ C05.getNewsletter(id) вҶ’ 404 if not found
  вҶ’ S3.deleteObject({ key: 'drafts/<filename>.md' }) вҖ” ignore NoSuchKey error
  вҶ’ S3.deleteObject({ key: 'published/<filename>.html' }) вҖ” ignore NoSuchKey error
  вҶ’ C05.deleteNewsletter(id)
  вҶ’ return 204
```

## 7. SAP Object Store Configuration

The S3 client is configured from environment variables. All values are required вҖ” startup fails fast if any are missing.

| Env var | Purpose |
| :------ | :------ |
| `OBJECT_STORE_ACCESS_KEY_ID` | S3 access key ID |
| `OBJECT_STORE_SECRET_ACCESS_KEY` | S3 secret access key |
| `OBJECT_STORE_BUCKET` | S3 bucket name |
| `OBJECT_STORE_REGION` | S3 region (e.g. `eu-central-1`) |
| `OBJECT_STORE_HOST` | S3 endpoint hostname (e.g. `s3-eu-central-1.amazonaws.com`) |

The `@aws-sdk/client-s3` client is initialised with `endpoint: https://<OBJECT_STORE_HOST>` and `forcePathStyle: true` to ensure S3-compatible path-style URLs.

---

## 8. Execution Mode

- All operations are synchronous (async/await). No background workers.
- `saveDraft()` is called in-process by C02 at the end of the generation pipeline вҖ” it is not a REST endpoint, it is an exported TypeScript function.
- All REST endpoints complete within the Express request lifecycle.

## 9. Error Handling

| Scenario | HTTP status | Error code | Behaviour |
| :------- | :---------: | :--------- | :-------- |
| Newsletter not found | 404 | `LIFECYCLE_NOT_FOUND` | Standard not-found |
| Already published (edit action) | 409 | `LIFECYCLE_ALREADY_PUBLISHED` | Cannot edit a published newsletter |
| Already published (publish action) | 409 | `LIFECYCLE_ALREADY_PUBLISHED` | No-op with clear message |
| Preview requested on published newsletter | 409 | `LIFECYCLE_NOT_DRAFT` | Direct to `/published/` URL instead |
| Object Store write failure | 502 | `LIFECYCLE_STORE_WRITE_FAILED` | HANA record not created/updated; error returned |
| Object Store read failure | 502 | `LIFECYCLE_STORE_READ_FAILED` | Error returned to caller |
| Markdown conversion failure | 500 | `LIFECYCLE_CONVERT_FAILED` | Object Store write not attempted; draft record unchanged |
