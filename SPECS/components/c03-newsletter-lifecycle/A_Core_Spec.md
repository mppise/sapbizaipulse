# C03 Newsletter Lifecycle — Core Specification

> Manages the draft → published lifecycle for newsletters. Saves generated `.md` drafts to SAP Object Store, converts to `.html` on publish, stores the published file in Object Store, and records all state in HANA Cloud via C05. No AI capabilities.

---

## 1. Purpose

C03 is the custodian of the newsletter lifecycle after generation. It:

- Receives a generated Markdown string from C02 and saves it as a `.md` draft to SAP Object Store; records the newsletter record in HANA Cloud via C05.
- Exposes a list/detail API for the Newsletter tab in the React UI.
- On publish: retrieves the `.md` from Object Store, converts it to HTML via `markdown-it`, writes the `.html` to Object Store, updates the newsletter record in C05 (`status = published`, `published_at`, `object_store_key` updated to HTML key).
- Serves the published HTML directly from Object Store via a signed or proxied URL, or by streaming the Object Store object through Express.
- Supports deletion of a newsletter (removes both Object Store objects and the HANA record).

## 2. Features

| Status | ID | Description | Priority | Doc Level |
| :----: | :- | :---------- | :------: | :-------: |
| `Complete` | F-C03-SAVEDRAFT | Receive a newsletter filename, topic list, and Markdown content string; write the `.md` file to Object Store under key `drafts/<filename>.md`; insert a newsletter record in C05 with `status = draft` and `object_store_key = drafts/<filename>.md` | P0 | - |
| `Complete` | F-C03-LIST | Return a list of all newsletter records (metadata only) ordered by `created_at` descending | P0 | Page |
| `Complete` | F-C03-DETAIL | Return full detail for a single newsletter record including `object_store_key` and `topic_list` | P0 | - |
| `Complete` | F-C03-PREVIEW | Retrieve the `.md` content from Object Store for a draft newsletter and return it as a string for UI preview rendering | P0 | Component |
| `Complete` | F-C03-EDIT | Accept an updated Markdown string for a draft newsletter; overwrite the `.md` file in Object Store under `drafts/<filename>.md`; draft-only — reject with 409 if already published | P0 | Component |
| `Complete` | F-C03-PUBLISH | Convert a draft newsletter to HTML: retrieve `.md` from Object Store → convert via `markdown-it` → write `.html` to Object Store under key `published/<filename>.html` → update C05 record (`status = published`, `published_at = now`, `object_store_key = published/<filename>.html`) | P0 | Component |

## 3. Newsletter Lifecycle State Machine

```
         [C02 calls saveDraft()]
                  │
                  ▼
           ┌──────────┐
           │  draft   │
           └────┬─────┘
                │ author publishes (F-C03-PUBLISH)
                ▼
       [retrieve .md from Object Store]
                │
       [convert .md → .html via markdown-it]
                │
       [write .html to Object Store]
                │
       [C05.updateNewsletter(status=published)]
                │
                ▼
         ┌───────────┐
         │ published │
         └───────────┘
```

**Rules:**
- A newsletter starts as `draft` when C02 calls `saveDraft()`.
- `published` is a terminal state — there is no unpublish action at MVP.
- Deletion is allowed from both `draft` and `published` states.
- A published newsletter's HTML is publicly accessible via `GET /published/:filename.html` — no API key required.

## 4. Object Store Key Convention

| File type | Object Store key pattern | Notes |
| :-------- | :----------------------- | :---- |
| Draft Markdown | `drafts/<filename>.md` | Written by `saveDraft()`; filename generated by C02 (e.g. `newsletter_2026-05-03`) |
| Published HTML | `published/<filename>.html` | Written by `publish()`; same base filename as the draft |

The `object_store_key` column in the `newsletters` HANA record always reflects the **current primary file** — `drafts/...` for draft, `published/...` for published.

## 6. Data Flows

### 6.1 Save draft (called by C02 after generation)

```
saveDraft({ filename, topicList, markdownContent })
  → S3.putObject({ key: 'drafts/<filename>.md', body: markdownContent })
  → C05.insertNewsletter({ filename, topicList, objectStoreKey: 'drafts/<filename>.md' })
  → return { id, filename, status: 'draft' }
```

### 6.2 List / detail

```
GET /api/v1/newsletters
  → C05.listNewsletters()
  → return array of newsletter metadata

GET /api/v1/newsletters/:id
  → C05.getNewsletter(id) → 404 if not found
  → return newsletter record
```

### 6.3 Edit draft content

```
PUT /api/v1/newsletters/:id/content
  → C05.getNewsletter(id) → 404 if not found; 409 if status = published
  → S3.putObject({ key: 'drafts/<filename>.md', body: markdownContent })
  → return 200 { id, filename }
```

### 6.4 Preview draft

```
GET /api/v1/newsletters/:id/preview
  → C05.getNewsletter(id) → 404 if not found; 409 if status = published
  → S3.getObject({ key: record.objectStoreKey })
  → return { markdownContent: '<string>' }
```

### 6.4 Publish

```
POST /api/v1/newsletters/:id/publish
  → C05.getNewsletter(id) → 404 if not found; 409 if already published
  → S3.getObject({ key: 'drafts/<filename>.md' }) → retrieve Markdown
  → markdown-it.render(markdownContent) → htmlContent
  → S3.putObject({ key: 'published/<filename>.html', body: htmlContent, contentType: 'text/html' })
  → C05.updateNewsletter(id, { status: 'published', publishedAt: now, objectStoreKey: 'published/<filename>.html' })
  → return { id, filename, status: 'published', publishedAt, htmlPath: '/published/<filename>.html' }
```

### 6.5 Serve published HTML

```
GET /published/:filename.html  (unauthenticated)
  → C05.listNewsletters() filtered by filename → 404 if not found or not published
  → S3.getObject({ key: 'published/<filename>.html' })
  → pipe S3 object stream to Express response (Content-Type: text/html)
```

### 6.6 Delete

```
DELETE /api/v1/newsletters/:id
  → C05.getNewsletter(id) → 404 if not found
  → S3.deleteObject({ key: 'drafts/<filename>.md' }) — ignore NoSuchKey error
  → S3.deleteObject({ key: 'published/<filename>.html' }) — ignore NoSuchKey error
  → C05.deleteNewsletter(id)
  → return 204
```

## 7. SAP Object Store Configuration

The S3 client is configured from environment variables. All values are required — startup fails fast if any are missing.

| Env var | Purpose |
| :------ | :------ |
| `OBJECT_STORE_ACCESS_KEY_ID` | S3 access key ID |
| `OBJECT_STORE_SECRET_ACCESS_KEY` | S3 secret access key |
| `OBJECT_STORE_BUCKET` | S3 bucket name |
| `OBJECT_STORE_REGION` | S3 region (e.g. `eu-central-1`) |
| `OBJECT_STORE_HOST` | S3 endpoint hostname (e.g. `s3-eu-central-1.amazonaws.com`) |

The `@aws-sdk/client-s3` client is initialised with `endpoint: https://<OBJECT_STORE_HOST>` and `forcePathStyle: true` to ensure S3-compatible path-style URLs.

---

## 8. Execution Mode

- All operations are synchronous (async/await). No background workers.
- `saveDraft()` is called in-process by C02 at the end of the generation pipeline — it is not a REST endpoint, it is an exported TypeScript function.
- All REST endpoints complete within the Express request lifecycle.

## 9. Error Handling

| Scenario | HTTP status | Error code | Behaviour |
| :------- | :---------: | :--------- | :-------- |
| Newsletter not found | 404 | `LIFECYCLE_NOT_FOUND` | Standard not-found |
| Already published (edit action) | 409 | `LIFECYCLE_ALREADY_PUBLISHED` | Cannot edit a published newsletter |
| Already published (publish action) | 409 | `LIFECYCLE_ALREADY_PUBLISHED` | No-op with clear message |
| Preview requested on published newsletter | 409 | `LIFECYCLE_NOT_DRAFT` | Direct to `/published/` URL instead |
| Object Store write failure | 502 | `LIFECYCLE_STORE_WRITE_FAILED` | HANA record not created/updated; error returned |
| Object Store read failure | 502 | `LIFECYCLE_STORE_READ_FAILED` | Error returned to caller |
| Markdown conversion failure | 500 | `LIFECYCLE_CONVERT_FAILED` | Object Store write not attempted; draft record unchanged |
