# C03 Newsletter Lifecycle — Specialized Specifications

> No AI integration. Covers NFRs, Object Store client setup, and markdown-it configuration.

## 1. Non-Functional Requirements

| ID | Requirement | Threshold | Measurement | Priority |
| :- | :---------- | :-------- | :---------- | :------: |
| NFR-C03-PUBLISH | Publish conversion (md → html) and Object Store write must complete | < 5s | Measured from `POST /publish` receipt to response sent | P0 |
| NFR-C03-SERVEHTML | Published HTML must be accessible immediately after the publish response | Verified by a GET to `/published/:filename.html` immediately after publish completes | Manual smoke test | P0 |
| NFR-C03-IDEMPOTDEL | Object Store deletes must be idempotent — deleting a non-existent key must not throw | NoSuchKey silently ignored | Code review | P0 |
| NFR-C03-NOLOCAL | No newsletter file content may be written to the local pod filesystem | 0 local file writes for .md/.html | Code review | P0 |

## 2. Object Store Client Configuration

`@aws-sdk/client-s3` is instantiated with the following options:

```typescript
new S3Client({
  region: process.env.OBJECT_STORE_REGION,
  endpoint: `https://${process.env.OBJECT_STORE_HOST}`,
  forcePathStyle: true,   // required for S3-compatible non-AWS endpoints
  credentials: {
    accessKeyId: process.env.OBJECT_STORE_ACCESS_KEY_ID,
    secretAccessKey: process.env.OBJECT_STORE_SECRET_ACCESS_KEY,
  },
});
```

`forcePathStyle: true` is mandatory — SAP Object Store uses path-style URLs (`https://host/bucket/key`), not virtual-hosted-style (`https://bucket.host/key`).

The bucket name is read from `OBJECT_STORE_BUCKET` and passed to every S3 command as the `Bucket` parameter. It is not embedded in the endpoint URL.

## 3. Markdown-it Configuration

```typescript
import MarkdownIt from 'markdown-it';

const md = new MarkdownIt({
  html: false,       // do not pass raw HTML through (security default)
  xhtmlOut: false,
  breaks: false,
  linkify: true,     // auto-linkify plain URLs in text
  typographer: false,
});
```

`html: false` ensures that any raw HTML that may have been injected into the generated Markdown by the LLM is escaped rather than rendered. This is a security default consistent with D-SEC-APIKEY and the architecture's output encoding guidance (B_Architecture.md §11.5).

## 4. Filename Convention

The `filename` field (e.g. `newsletter_2026-05-03`) is generated by C02 at generation time. C03 appends the extension:
- Draft key: `drafts/<filename>.md`
- Published key: `published/<filename>.html`

C03 does not generate filenames — it accepts whatever filename C02 provides via `saveDraft()`. If C02 generates a duplicate filename, the Object Store `putObject` call will silently overwrite the existing object. C03 does not perform a uniqueness check on the filename — that responsibility belongs to C02.
